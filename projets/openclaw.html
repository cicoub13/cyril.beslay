OpenClaw.

Un agent Claude self-hosted, scriptable, qui tourne en local sur mon homelab. Plugins maison, mémoire persistante, accès aux services Zigbee et au NAS. Pas de bureau distant, pas de SaaS.

TypeScript Bun SQLite Anthropic API systemd mTLS
— Contexte

Je voulais un assistant capable de toucher à mon homelab : couper la lumière du bureau, regarder les logs d'Home Assistant, lire un PDF posé sur le NAS, lancer une sauvegarde. Aucun produit SaaS ne pouvait faire ça sans envoyer la moitié de ma maison dans un cloud lointain.

OpenClaw est ma réponse : un petit daemon qui appelle l'API Anthropic, expose des outils maison, et garde ses traces en local.

— Approche

Le cœur est minuscule : ~1500 lignes de TypeScript, un loop d'outils, une mémoire SQLite. Tout le reste est en plugins, branchés via convention de dossier.

L'accent est mis sur l'observabilité — chaque appel d'outil est loggé, replayable, et inspectable depuis une mini-UI web servie en local.

  • Outils maison : Home Assistant, fichiers NAS, calendrier CalDAV, shell sandbox
  • mTLS pour l'accès depuis le téléphone via Tailscale
  • Système de “profils” pour cloisonner les permissions par contexte (cuisine, bureau, mobile)
  • Mémoire vectorielle légère, embeddings calculés en local
— Lessons learned

Le piège c'est de vouloir tout abstraire. La v1 avait un système de plugins trop générique, j'ai simplifié en v0.3 et je le regrette pas.

Self-host un agent c'est facile, lui faire confiance c'est autre chose. Le permissioning par profil change tout, vraiment.
